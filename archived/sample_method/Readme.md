作者在论文及附录中对超参数的选择有非常明确的指导，并且这些选择直接与他们对有向图特征的数学洞察（Insights）绑定。对比你代码中的默认设置，以下是作者点名强调的核心参数及背后的底层逻辑：

### 1. Magnetic Laplacian 的相对势能（Relative Potential $q'$）
* **作者参数**：推荐使用 $q' \in \{0.1, 0.25\}$，在大多数实验（如排序网络和 OGB Code2）中固定为 $q' = 0.25$ [cite: 146, 1277]。
* **作者 Insight**：
    * 直接使用固定的绝对势能 $q$ 会导致在大型图中相位偏移过大，产生高频振荡，从而失去对局部结构的表征能力 [cite: 1166, 1167]。
    * 作者提出必须使用图相关的归一化：$q = q' / d_{\mathcal{G}}$，其中 $d_{\mathcal{G}} = \max(\min(\vec{m}, n), 1)$，$\vec{m}$ 是纯有向边的最大数量 [cite: 145]。这种设计保证了从源节点到目标节点的最大相位差被限制在合理范围内（避免退化情况） [cite: 1085, 1167]。

### 2. 特征向量的数量（$k$ for ML）
* **作者参数**：在所有的主要实验（Playground、排序网络、OGB Code2）中，作者均设定保留前 **$k=25$** 个特征向量 [cite: 1277]。
* **对比与修正**：你的代码中默认设置了 `pe_k=5`。如果计算资源允许，建议将其提升至 25，以捕获足够多的图频率信息 [cite: 1277]。

### 3. Random Walk 的步数（$k$ for RW）与 PPR
* **作者参数**：明确指出随机游走步数 $k=2$ 或 $k=3$ 已经足够 [cite: 1429, 1277]。同时，Personalized PageRank (PPR) 的重启概率设定为 $p_r = 0.05$ [cite: 1277]。
* **作者 Insight**：
    * 对于有向图，如果 $k$ 过大，概率质量（Probability mass）会迅速集中到 Sink 节点（即没有出度的节点），导致位置编码失效 [cite: 212]。因此有限步游走只适合捕获短距离结构 [cite: 213]。
    * 必须引入带有重启概率 $p_r$ 的 PPR 来补偿长距离（全局）关系 [cite: 214, 215]。

### 4. SignNet 与 MLP 的条件选择
* **作者 Insight**：这是论文中一个非常反直觉的结论。
    * **无 GNN 场景**：在使用纯 Transformer 时，如果特征向量按规范化处理了符号，直接使用逐元素的 MLP 会比使用 SignNet 效果更好 [cite: 188, 1420]。作者发现 SignNet ($f(x)+f(-x)$) 的行为类似于取绝对值，这会丢失特征向量中包含符号变化的相对差异信息 [cite: 1422, 1423]。
    * **有 GNN 场景（契合你的代码）**：在类似 OGB Code2 这样引入了局部消息传递（GNN）的混合架构中，SignNet 确实能帮助提升性能 [cite: 1424]。你的模型包含 `GraphConvolutionLayer`，保留 SignNet 是合理的选择。
